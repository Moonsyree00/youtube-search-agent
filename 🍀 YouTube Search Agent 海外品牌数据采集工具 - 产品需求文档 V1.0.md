# 🍀 YouTube Search Agent 海外品牌数据采集工具 \- 产品需求文档 V1\.0

# 前言

本 PRD 为独立开发项目《YouTube Search Agent 海外品牌数据采集工具》V1\.0 版本需求文档，面向科研团队无代码基础研究员设计，完整定义工具定位、用户痛点、全链路功能规范、数据输出标准与验收规则。

## 文档目标

1. 清晰梳理产品整体业务逻辑、分层架构与交互规则，作为项目开发落地依据；

2. 标准化海外 YouTube 品牌数据采集流程，统一数据集输出格式，满足商科品牌出海实证科研的数据溯源、量化分析需求；

3. 明确零代码 GUI 交互、大模型 Agent 智能筛选、API 批量采集、容错断点机制等核心能力边界；

4. 留存完整产品设计思路，用于求职数据 / 商业分析类岗位面试复盘，完整展示数据自动化工具全流程设计与工程落地思维。

## 阅读对象

2. 团队科研成员（工具实际使用者，快速看懂工具能力、使用流程）

1. 本文档仅覆盖 V1\.0 落地范围，TikTok、Instagram 多渠道拓展、在线 Web 部署等能力暂不纳入本版本；

# 四、 需求背景

## 产品 / 数据现状

当前团队开展品牌出海海外社交媒体实证研究，YouTube 渠道数据获取存在完整流程痛点：

1. 团队多数科研人员无 Python、API 开发基础，无法独立运行爬虫脚本，每次采集数据均需技术人员协助，沟通排期成本高；

2. 纯人工检索品牌频道易混淆官方主账号、区域代理号、粉丝自媒体号，人工逐条核验、筛选效率低，上百个品牌批量处理需花费大量工时；

3. 零散独立爬虫脚本无统一工程规范，无断点保存机制，程序崩溃后已采集数据易丢失；输出文件无标准化字段，每次实证前需要手动清洗、整理表格；

4. 缺少可视化操作界面，批量导入品牌清单、时间段筛选、日志查看等操作均需修改代码参数，使用门槛高。

## 用户调研

一对一访谈团队内4名商科科研研究员，覆盖本科、硕士阶段品牌出海课题研究者，同时结合日常数据取数需求复盘，归纳统一痛点诉求。

1. 核心用户：无代码科研研究员

- 特征：懂实证分析、会 Excel/SPSS，不会 Python、接口调试；

- 核心诉求：零代码可视化操作，上传品牌 CSV、自然语言下发任务即可批量采集；输出标准化 CSV 数据集，可直接用于回归建模；AI 自动筛选官方频道，减少人工复核。

2. 次要用户：技术辅助人员

- 特征：掌握基础 Python，负责批量批量调度、批量实验；

- 核心诉求：提供 CLI 命令行模式，外置配置文件统一管理密钥，支持脚本自动化批量跑任务。

🌟 调研结论

团队亟需一套低门槛、自动化、标准化的海外 YouTube 数据采集工具，通过大模型 Agent 替代人工筛选、封装 API 实现批量采集、配套 GUI 降低使用门槛，统一科研数据输出规范，大幅减少人工检索、清洗、核对的重复工作。

# 七、 非功能需求

## 性能需求

1. 单品牌频道完整检索、AI 判别平均耗时 ≤3s；

2. GUI 流式对话输出延迟 ≤200ms，长时间批量采集页面无卡顿、无假死；

3. YouTube API 分页请求间隔 0\.5s，内置限流缓冲机制，稳定避免 429 报错；

4. 单批次支持一次性处理至少 100 条品牌清单，文件读写无阻塞。

## 兼容性需求

1. 系统：完整支持 Windows 全版本本地离线运行；

2. 大模型接口：原生兼容 OpenAI 标准接口，同时适配 DeepSeek、通义千问等国产兼容格式 API；

3. 输出文件：统一 utf\-8\-sig 编码 CSV，可直接用 Excel、SPSS、Python、WPS 打开无中文乱码；

4. 输入文件：支持中英文表头混合 CSV 自动归一识别。

## 容错与稳定性需求（FDE 重点加分项）

1. 断点续存：单个品牌处理完成即时写入 CSV，程序崩溃、手动关闭不会丢失已处理数据；

2. 异常隔离：单条品牌 API 失败、模型调用报错仅跳过当前品牌，不中断整体批量任务；

3. 全链路前置校验：空文件、无效品牌名、过期 API 密钥、错误时间区间提前拦截并给出文字提示；

4. API 自动重试：网络波动、接口超时自动重试 3 次，重试失败记录错误字段存入输出表格；

5. GUI 交互锁：任务执行中锁定发送、文件上传按钮，防止重复提交多任务冲突。

## 可拓展需求

1. 渠道拓展：新增 TikTok/Instagram 采集渠道仅需新增 API 工具函数，注册至 Agent 调度层即可，无需改动核心业务流程；

2. 输出拓展：可快速新增 Excel、本地数据库入库等存储方式，统一复用现有数据清洗模块；

3. 模型拓展：新增大模型服务商仅需修改 config\.json 配置文件，无需修改业务代码。

## 安全需求

1. 密钥隔离：YouTube API Key、大模型密钥统一存放于外置 config\.json，不硬编码嵌入代码；

2. 本地离线：所有采集数据集、日志文件仅保存在用户本地设备，无自动上传云端逻辑；

3. 无第三方数据中转：API 请求直连官方接口，不通过中间代理服务转发敏感参数

# 八、验收标准

## GUI 客户端验收

1. 上传 100 条品牌 CSV，程序无卡顿、无崩溃，进度条实时更新；

2. 任务运行时上传 / 发送按钮置灰，重复点击无重复任务；

3. 未完成任务关闭窗口，弹出确认弹窗。

## Agent 调度验收

1. 输入任意品牌名称，Agent 按「检索→核验→采集」顺序调用工具，不会乱序；

2. 切换 3 种不同大模型服务商，均可稳定输出结构化判定 JSON，无解析失败；

3. 单个品牌模型报错仅跳过本条，不中断整批任务。

## 采集工作流验收

1. API 连续分页请求无 429 限流报错，失败自动重试 3 次；

2. 中途关闭程序，重新启动可继续处理剩余未完成品牌；

3. 输出 CSV 字段与附录规范完全匹配，无缺失、乱码。

## 兼容性验收

1. Windows10/11 均可离线完整运行；

2. 导出 CSV 可直接用 Excel、SPSS、Python 读取无中文乱码。

## 安全验收

1. 所有密钥存储在 config\.json，全局代码检索无明文硬编码密钥；

2. 所有采集文件、日志仅本地存储，无任何自动上传云端逻辑。

# 附录：数据集输出字段规范

本附录定义工具核心输出的 2 类标准化 CSV 文件的字段规范，所有文件统一采用`utf-8-sig`编码，兼容 Excel、SPSS、Python 等工具直接读取，满足科研数据溯源与实证分析需求。

## 品牌频道匹配主结果表

**文件名格式：**`brand_channel_ids_YYYYMMDD_HHMMSS.csv`

**核心用途：**批量品牌官方 YouTube 频道核验的核心基准表，记录每个品牌的最终匹配状态、核心运营指标、AI 判定依据与异常信息，是后续批量视频采集的核心输入数据源，也是品牌账号渠道筛选的最终成果表。

## 单品牌视频明细数据表

**文件名格式：**`{品牌名}_videos_YYYYMMDD_HHMMSS.csv`

**核心用途：**工具核心产出数据集，存储指定品牌下所有视频的全量元数据，是海外品牌营销实证分析、内容效果建模、竞争力量化研究的核心数据源，可直接导入统计软件完成回归、聚类、中介效应等分析。

## 品牌频道全量信息表

**文件名格式：**`brand_channel_info_YYYYMMDD_HHMMSS.csv`

**核心用途：**批量品牌频道核验的全量结果存储表，与主结果表结构完全一致，用于批量处理全量品牌的频道信息归档、多批次核验结果对比、品牌渠道画像全量存储，支持大样本品牌渠道特征聚类分析。

## 补充说明

1. 样例数据报错说明：当前提供的样例数据中频道表全量`error`状态，是 YouTube Data API 密钥权限不足 / 配额耗尽导致的接口调用失败，并非表结构设计问题，更换有效 API 密钥即可正常获取全量频道数据；

2. 文件命名规范：所有输出文件均自动附加批次时间戳，避免文件覆盖，支持多批次采集结果的区分与归档。
